从Python装饰器初识函数式编程

发表于 2021-02-16 | 分类于 语言 |
字数统计: 2.2k

前言

很多人估计和我一样一开始看到Python装饰器会想到Java里的注解或者设计模式里的装饰器模式。实际上三者相差很大，
尽管都是为了在原有的方法或者模块上新增一些小功能，Java注解基于各种类库，类似一种元数据，而装饰器模式基于OO面向对象。
Python装饰器本质上只是一个语言层面的概念，就是函数，所以也算是函数式编程中的一种技巧。

装饰器的概念与使用

基本前提：函数在Python里面属于一等公民，函数也是对象。所以在Python里面函数也可以像普通的对象一样作为参数传递给其他函数，
可以被赋值给其他变量，可以作为返回值，也可以被定义在另一个函数内(嵌套函数)。
当然装饰器也可以通过类的定义实现，本文主要讲函数式实现

函数式编程

函数式编程属于一种编程范式，常见的编程范式有命令式编程(包括面向对象)、函数式编程、逻辑式编程。
命令式编程是一种面向计算机硬件的抽象编程范式，函数式编程是面向数学的抽象，将计算描述为表达式求值。
函数式编程和命令式编程有各自特性和优缺点，这里参考网友@nameoverflow的解释，两者之间最大的区别在于：

函数式编程编程关注数据的映射，命令式编程关注解决问题的步骤

函数式编程本质

函数式编程通过在函数中定义表达式和对表达式求值完成计算。尽量避免使用可变对象或者涉及状态变化是程序复杂化。
这里的函数指的是数学上的函数概念：自变量的映射。一个函数求值不管你怎么调用，只取决于函数参数，与其他状态无关。
例如求平方根函数：sqrt(x),这个函数就是对变量x的一种映射关系，x不变，sqrt(x)就不会变。
拿求平方根举例，下面的代码写法你们觉得属于函数式编程吗？用了二分查找的思路，二分查找时通过比较mid的平方与x的大小关系来调整上下界范围。
这种写法并不是函数式思维的写法，描述的是怎么一步步二分求出平方根，而不是描述一个res -> x的映射

如果用牛顿迭代法怎么写一个函数式的求平方根
牛顿迭代法通过一个起始值和一个关系式一步步向结果收敛：

$$a_{i+1} = (a_i + n / a_i) / 2 $$

如果近似值收敛到某个极限a

$$a = (a + n/a) / 2 $$

$$2a = a + n / a $$

$$a^2 = n $$

$$a = \sqrt{n}$$

python在函数式方面的一些应用

从上面计算sqrt函数的设计用到了python迭代器和生成器的概念，这两个家伙是编写函数式风格程序的重要基石。
实际上python，C++这种都是多范式语言，并非纯函数式语言，但也支持或者说拥有一些常用的函数式编程功能：

  • map、reduce、filter

  • pipeline

  • 递归

  • 高阶函数：高阶函数就是把函数当做参数传来穿去，返回来返回去

  • 闭包：闭包就是能够读取其他函数内部变量的函数。

所以上面讲的python装饰器本质也是高阶函数，也是闭包
